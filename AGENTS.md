# AGENTS.md

## Project
校园网络设备管理与故障报修系统

技术栈：
- Backend: Spring Boot + MyBatis-Plus + JWT + RBAC + MySQL
- Frontend: Vue3 + Vite + Element Plus + Pinia + Vue Router

## Core objective
本项目不是演示型后台，而是企业级业务系统。
所有修改必须优先满足：
1. 可实际流转
2. 可跟踪
3. 可统计
4. 可留痕
5. 可审计
6. 前后端真实联动
7. 数据结构与业务逻辑一致

## Mandatory business rules
1. 角色固定为三类：
   - 系统/业务管理员
   - 维修人员
   - 报修用户

2. 所有用户必须有工号 `employeeNo`
   - 全局唯一
   - 新增、编辑、导入都必须校验唯一性
   - 登录、查询、筛选必须支持工号

3. 登录必须满足：
   - 先进入门户首页
   - 先选角色，再登录
   - 登录表单包含：工号/用户名、密码、验证码
   - 预留微信/QQ登录入口和后端占位接口
   - 登录失败必须区分原因并记录日志

4. 修改密码必须满足：
   - 旧密码
   - 新密码
   - 确认新密码
   - 验证码
   - 两次新密码一致
   - 密码强度校验
   - 成功失败都要明确提示和记录日志

5. 工单必须是完整生命周期，不允许简化成普通 CRUD。
   至少支持状态：
   - 待提交
   - 已提交/待审核
   - 审核通过/审核驳回
   - 待分配
   - 已分配
   - 待接单
   - 已接单
   - 维修中
   - 待采购/待配件
   - 申请延期中
   - 延期已批准
   - 待验收/待确认
   - 已完成
   - 已关闭
   - 已取消

6. 工单流转必须校验：
   - 未审核不能分配
   - 未分配不能接单
   - 未接单不能开始维修
   - 未完成不能待验收
   - 用户未确认前不能普通关闭
   - 强制关闭必须记录原因
   - 终态工单不能随意改核心字段

7. 所有关键动作必须同时记录：
   - 工单流程记录 flow
   - 业务日志 business_log
   - 必要时系统审计日志 operation_log

8. 设备管理必须是完整档案，不允许只有表格。
   设备详情页至少包含：
   - 基本信息卡片
   - 当前状态
   - 维修次数
   - 最近维修记录
   - 维修时间线
   - 故障原因统计
   - 维修照片
   - 关联工单
   - 保修状态
   - 高故障预警
   - 是否建议更换/巡检

9. 图片相关必须走真实上传，不允许用户手填图片 URL 作为正式方案。
   需要：
   - 后端 multipart 上传接口
   - 文件元数据入库
   - 可访问 URL
   - 前端上传与预览

10. 统计分析必须是业务统计，不是装饰图表。
    至少支持：
    - 日/月/半年/年/自定义时间范围
    - 报修数、完成数、未完成数、平均维修时长
    - 设备类型排行
    - 高频故障设备排行
    - 故障原因分布
    - 维修人员工单数和平均处理时长
    - 满意度统计
    - 延期占比
    - 配件采购占比
    - 预测修复时间/误差分析
    - 图表点击跳转明细

11. 公告不做普通用户单独菜单。
    - 首页直接展示公告
    - 管理员仍要有公告发布/管理入口

12. 所有新增功能必须同步修改：
    - SQL / schema
    - entity / dto / vo / mapper / service / controller
    - frontend api / route / page / permission
    不允许只改一端。

## Engineering rules
1. 优先修一致性问题，再补新功能。
2. 不保留双轨实现；如果 `module/v2/**` 与主线冲突，优先收口到主线。
3. 不允许出现：
   - 实体有字段但数据库无列
   - 前端调接口但后端没有
   - 页面文件存在但没有路由和菜单入口
4. 改代码时尽量保留可用部分，但可以重构不合理结构。
5. 不要写 TODO 假实现，不要只做静态页面。
6. 所有动作要有权限控制、参数校验、异常提示。
7. 先列出问题清单，再动手改代码。
8. 每次只处理一个阶段任务，不要一次改全项目。

## Expected output for each task
每次任务完成后必须输出：
1. 修改文件列表
2. 数据库改动
3. 后端接口改动
4. 前端页面改动
5. 自检结果
6. 剩余风险
