# 第4步：登录认证与验证码机制重构

## 本步完成

1. 登录门户页增强：系统名称、简介、适用场景、所属单位、公告联动、角色入口。
2. 登录流程改造为“两步”：先选角色，再进入对应角色登录表单。
3. 每种角色登录表单支持：工号/用户名、密码、验证码、登录按钮。
4. 保留并联调第三方登录预留（微信/QQ 前端按钮 + 后端占位接口）。
5. 登录失败原因细分并写入登录日志：
   - 角色选择错误
   - 账号不存在
   - 密码错误
   - 验证码错误
   - 账号被禁用
   - 非本角色账号禁止登录

## 本步接口变化

- `POST /api/auth/login`：增强失败原因区分。
- `GET /api/auth/captcha`：继续作为验证码接口使用。
- `GET /api/auth/oauth/{provider}/url`：第三方登录占位。
- `GET /api/portal/home`：新增 `scenes`、`loginEntries` 字段。

## 本步新增页面

- 无新增独立路由页面。
- `PortalView` 与 `LoginView` 已重构为完整门户 + 两步式角色登录体验。

## 本步待后续

- 真实图形验证码/短信验证码能力。
- 微信/QQ OAuth 真正接入（当前为占位逻辑）。
- 登录风控（失败次数限制、锁定策略、多端会话管理）。
