# 第5步：密码修改与安全校验重构

## 本步完成

1. 修改密码接口重构为 DTO 入参：旧密码、新密码、确认新密码、验证码、验证码Key。
2. 后端增加旧密码校验、验证码校验、两次新密码一致性校验。
3. 后端增加密码强度规则：
   - 长度至少 8 位
   - 必须包含字母和数字
4. 前端修改密码页补充：
   - 旧密码
   - 新密码
   - 再次输入新密码
   - 验证码
   - 强度提示文案
5. 修改成功/失败均给出明确提示。
6. 修改密码成功/失败均写入操作日志（包含失败原因）。

## 本步接口变化

- `PUT /api/auth/updatePassword` 入参变更为 `UpdatePasswordDTO`。

## 本步新增页面

- 无新增路由页面。
- `ProfileView` 中“修改密码”模块已重构。

## 本步待后续

- 增加密码历史校验（禁止近期历史密码复用）。
- 增加强密码等级评分展示（弱/中/强）。
- 增加密码修改后二次认证策略（如短信/邮件确认）。
