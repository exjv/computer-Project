# refactor-plan.md

## 阶段 1：基础一致性修复
目标：
- 修 schema / entity / mapper / permission / route / dead api 的不一致
- 让项目先稳定，不再频繁报字段或表不存在错误

范围：
1. role.remark
2. permission / role_permission 权限模型统一
3. NetworkDevice / RepairOrder / FileAttachment / ThirdPartyBind 字段对齐
4. 统一 SQL 初始化脚本
5. 收口 module/v2
6. 修复前端死接口
7. 重装前端依赖并验证 build

## 阶段 2：角色与权限交付
目标：
- 管理员可真正管理角色与权限
- 前端有正式角色管理页

范围：
1. 角色分页、增删改、启停
2. 角色权限展示/维护
3. 路由、菜单、权限码对齐

## 阶段 3：登录与门户重构
目标：
- 门户首页、角色选择登录、验证码、第三方登录预留、公告首页展示

范围：
1. 门户首页
2. 角色选择登录
3. 登录失败细分提示
4. 公告首页展示
5. 第三方登录预留架构整理

## 阶段 4：工单生命周期强化
目标：
- 工单真正成为工作流系统

范围：
1. 状态机收敛
2. 审核、分配、接单、维修、延期、待配件、待确认、关闭
3. flow + business_log 完整联动
4. 非法流转和终态保护

## 阶段 5：设备与上传留痕
目标：
- 设备详情企业化
- 图片真实上传

范围：
1. 设备详情卡片与统计
2. 设备维修时间线
3. 真实文件上传
4. 照片预览与附件管理

## 阶段 6：统计、报表、通知闭环
目标：
- 统计分析中心、导出、消息通知、满意度闭环

范围：
1. 独立统计页
2. Excel 导出整理
3. 消息中心
4. 满意度与返修闭环
5. 演示数据重构
