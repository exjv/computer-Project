# 论文辅助材料

## 1. E-R关系文字说明
系统核心实体包括：用户、设备、报修工单、维修记录、公告、操作日志、登录日志。
- 用户与报修工单：一对多（报修人）。
- 用户与报修工单：一对多（维修人员被分配）。
- 设备与报修工单：一对多。
- 报修工单与维修记录：一对多。
- 用户与公告：一对多（发布者）。
- 用户与日志：一对多。

## 2. 系统功能模块划分说明
- 权限认证模块：JWT登录与RBAC。
- 用户管理模块：管理员维护系统账号。
- 设备管理模块：设备全生命周期信息维护。
- 报修工单模块：用户提交、管理员分配、维修员处理。
- 维修记录模块：维修过程与结果沉淀。
- 公告模块：运维通知发布。
- 日志模块：登录与关键操作追溯。
- 统计模块：按角色展示关键指标。

## 3. 系统业务流程说明
用户登录后按角色进入不同菜单；普通用户提交报修后工单进入“待处理”，管理员分配维修人员后变更“已分配”，维修人员接单处理转为“处理中”，填写维修记录后可置为“已完成”。

## 4. 登录流程说明
前端提交用户名/密码 -> 后端校验 -> 生成JWT返回 -> 前端本地保存Token -> 请求拦截器自动携带Token -> 后端过滤器解析Token并注入身份信息。

## 5. 报修处理流程说明
用户选择设备提交报修 -> 系统默认状态“待处理”，设备标记“故障” -> 管理员分配维修人员并置“已分配” -> 维修人员更新状态“处理中”并填写维修记录 -> 若解决则工单“已完成”且设备状态恢复“正常”，未解决可保持“处理中”。
