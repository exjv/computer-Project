# 第10步：报修用户侧工单动作实现

## 本步完成

1. 用户侧动作能力落地：
   - 发起/提交报修（新建报修申请页）
   - 撤销报修（仅允许可撤销状态）
   - 查看本人报修记录
   - 查看工单进度
   - 维修完成后确认“已解决/未解决”
   - 提交满意度评价与反馈意见
2. 后端动作增强：
   - `RepairOrderActionDTO` 新增满意度和反馈字段
   - 用户验收确认时写入 `repair_feedback`
   - 用户反馈提交后写入 `business_log`
   - “未解决”动作触发状态回退到“维修中”，形成返修闭环
3. 前端页面新增：
   - `报修申请页`：`/repair-apply`
   - `我的报修记录页`：`/my-repairs`
   - `工单进度跟踪页`：`/my-repairs/:id/progress`
   - `验收确认/满意度评价`弹窗（在我的报修记录页）

## 安全约束

- 用户详情、流程与动作接口均走后端用户身份校验，只允许本人查看和操作自己的工单。
- 工单撤销仍通过后端状态校验控制，仅允许可撤销状态执行。

## 验证建议

1. 以普通报修用户登录：
   - 打开“报修申请”提交工单；
   - 在“我的报修记录”中查看新工单；
2. 工单流转到“待验收/待确认”后：
   - 执行“已解决”提交评分和反馈；
   - 执行“未解决”并提交反馈，确认工单返回“维修中”；
3. 检查后端 `repair_feedback` 与 `business_log` 数据是否落库。
